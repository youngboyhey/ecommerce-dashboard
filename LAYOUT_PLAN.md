---

## 🧠 設計哲學：電商老闆的一天

電商老闆每天早上打開 Dashboard，心裡想的是：

1. **「今天賺了多少？」** → 營收、訂單
2. **「廣告錢花得值不值？」** → ROAS、CPA、花費
3. **「有沒有什麼問題要處理？」** → 警示通知
4. **「跟昨天/上週比怎麼樣？」** → 趨勢對比

基於此，我們設計 **「由上而下、由緊急到細節」** 的資訊層級。

## 🚨 Zone 1：警示區塊（Alert Banner）

### 設計理念
- **位置**：固定在最頂部，無法忽視
- **行為**：有警示時自動展開，無警示時收合為一行綠燈狀態
- **顏色**：🔴 紅色（嚴重）、🟡 黃色（注意）、🟢 綠色（正常）

### 警示觸發條件

| 指標 | 🟡 注意 | 🔴 嚴重 | 說明 |
|------|---------|---------|------|
| ROAS | < 2.0 | < 1.5 | 廣告投報率過低 |
| CPM | > 350 | > 400 | 千次曝光成本過高 |
| Frequency | > 2.0 | > 2.5 | 受眾疲勞，需換素材 |
| CPA | > 目標×1.2 | > 目標×1.5 | 獲客成本超標 |
| 網站跳出率 | > 60% | > 75% | 著陸頁需優化 |
| 購物車放棄率 | > 70% | > 85% | 結帳流程有問題 |

### 警示呈現格式
```
🔴 嚴重警示 (2)
├─ Meta Ads 廣告組「夏季促銷_新客」Frequency 達 3.2，建議更換素材
└─ 網站跳出率達 78%，著陸頁需優化

🟡 需注意 (1)  
└─ CPM 達 $380，接近警戒值
```

## 💰 Zone 2：核心 KPI 卡片區

### 設計理念
- **佔比**：螢幕最顯眼位置，一眼掃過就能掌握
- **數據**：只放「老闆最關心的 4 個數字」
- **對比**：必須有「跟昨天比」或「跟上週同期比」

### KPI 卡片內容

| 卡片 | 主數據 | 對比基準 | 顏色邏輯 |
|------|--------|----------|----------|
| 💰 營收 | 今日營收 | vs 昨日 / vs 上週同日 | ▲綠 ▼紅 |
| 📦 訂單 | 今日訂單數 | vs 昨日 / vs 上週同日 | ▲綠 ▼紅 |
| 📈 ROAS | 今日廣告投報率 | vs 昨日 / vs 7日均值 | ▲綠 ▼紅（低於1.5變紅） |
| 💸 CPA | 今日獲客成本 | vs 昨日 / vs 目標值 | ▲紅 ▼綠（CPA 低=好） |

### 進階互動
- 點擊卡片可展開「細節 Breakdown」
- 例：點營收 → 顯示各通路營收佔比

## 📊 Zone 3：趨勢與效率區

### 3A. 營收趨勢圖（RevenueTrendChart 強化版）

```
顯示內容：
├─ 近 7 天 / 30 天營收折線圖
├─ 疊加「廣告花費」作為對照
├─ 標註重要事件（促銷日、投放變動）
└─ 週末/平日標記不同背景色
```

### 3B. 廣告效率儀表板（MetaAdsChart 強化版）

```
區塊 A：效率指標儀表
├─ CPM 儀表（圓形/半圓，顯示當前值 vs 警戒值）
├─ Frequency 儀表（同上）
└─ Reach 累計進度條

區塊 B：Adset 層級表現表格
├─ 排序：預設按 ROAS 降序
├─ 欄位：Adset 名稱 | 花費 | 曝光 | CPM | Freq | 轉換 | ROAS
├─ 標記：ROAS < 1.5 的列標紅
└─ 動作按鈕：暫停 / 加碼（連結到 Meta Ads）
```

## 🔄 Zone 4：網站行為分析區

### 4A. GA4 轉換漏斗（GA4Funnel）

```
漏斗階段：
Session → 商品瀏覽 → 加入購物車 → 開始結帳 → 完成購買

顯示：
├─ 每階段人數 + 轉換率
├─ 與上週同期對比（箭頭標示漲跌）
└─ 點擊可展開該階段的流失分析
```

### 4B. 受眾分析（AudienceAnalysis 強化版）

```
Tab 1：新客 vs 回訪
├─ 圓餅圖：新客/回訪佔比
├─ 關鍵指標對比表
│   ├─ 平均訂單金額
│   ├─ 轉換率
│   └─ 頁面停留時間
└─ 洞察：「回訪客轉換率是新客的 2.3 倍」

Tab 2：流量來源
├─ 長條圖：各管道流量佔比
├─ 表格：管道 | 工作階段 | 轉換率 | 營收
└─ 標記：表現最佳 / 需優化的管道
```

### 4C. 裝置分布（新增組件：DeviceBreakdown）

```
顯示：
├─ 圓餅圖：Desktop / Mobile / Tablet 佔比
├─ 各裝置轉換率對比
├─ 互動指標（跳出率、頁面/工作階段、平均停留時間）
└─ 洞察：「Mobile 流量佔 72%，但轉換率僅 Desktop 的 0.6 倍」
```

## 🏆 Zone 5：商品與 SEO 區

### 5A. 商品銷售排行（ProductRanking）

```
顯示：
├─ Top 10 熱銷商品（今日/本週可切換）
├─ 欄位：排名 | 商品名 | 銷量 | 營收 | 庫存
├─ 庫存預警：庫存 < 10 標黃，< 5 標紅
└─ 變化箭頭：相較昨日排名變動
```

### 5B. SEO 表現（新增組件：GSCPerformance）

```
區塊 A：整體表現
├─ 總曝光數 | 總點擊數 | 平均 CTR | 平均排名
└─ vs 上週同期對比

區塊 B：關鍵字 Top 10
├─ 欄位：關鍵字 | 曝光 | 點擊 | CTR | 排名
├─ 排序：預設按點擊數降序
└─ 標記：排名進入前 3 的標綠

區塊 C：頁面表現 Top 10
├─ 欄位：頁面路徑 | 曝光 | 點擊 | CTR
└─ 快捷：點擊可跳轉 GA4 該頁面詳情
```

## ✅ 結論

這份 Dashboard 設計遵循「**3-10-30 法則**」：

| 時間 | 使用者能獲得 |
|------|-------------|
| 3 秒 | 今天賺了多少、有沒有問題 |
| 10 秒 | 廣告效率、流量變化 |
| 30 秒 | 商品表現、SEO 趨勢、深入分析入口 |

讓電商老闆每天打開 Dashboard，就像讀一份「營運健檢報告」——**重點先行，細節可查**。
